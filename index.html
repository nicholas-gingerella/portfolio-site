My Mission

My Engineering Oath

Working as a software engineer on many different teams has taught me important lessons that shape how I approach a project, and what values I feel are most important to a practitioner in this field.

Solve Problems

Ask questions and analyze the root cause of problems. Break down complex issues and create elegant solutions that provide lasting value, rather than short lived patches.

Value Quality

Have an eye for detail and usability when designing software, not only to present a more enjoyable product for your users, but to create a better developer experience for your peers.

Keep Learning

Learn from your mistakes and those around you, always seek knowledge and tools to boost your productivity and expand your skills. True learning requires effort, don't always take the easy path.

Work Together

Work with others to take on the big challenges you can't do alone. Through the teamwork of many contributing to a shared vision, we can all do great things.

Build for People

It's easy to become too focused on the technology itself, and forget about how the technology can be used to help people. Never forget, the technology can be novel, but if it doesn't solve a problem right now, then most people won't care much for it.

My Tools

Skills

I’m always on the hunt for new skills for my problem solving toolbox. I employ whatever languages and tools needed in order to get the job done.

C and C++

I use C and C++ for most of my system programming projects. I have experience building client server applications on linux operating systems using the Linux programming interface to support needs for concurrency, file IO, and more.

Python

I use Python for creating scripts to support automated testing and basic linux administration. I've also used Python for embedded development projects on Raspberry-Pi, and Web Development projects using Flask and Django.

HTML, CSS, and Javascript

The staples of web development, I use these for visually creative projects. I have experience working with the Bootstrap CSS framework and the JQuery javascript framework for creating websites and web applications.

Agile Development

When done right, Agile can help teams identify and focus on features that bring real value to their customers, while continuously improving their processes and workflow to enable everyone on the team to perform at their best.

Engineering Process

Mission critical projects require a well disciplined team with defined engineering processes. A good process not only helps reduce new bugs and regression, but also helps produce quality software design and test documentation.

Team Building & Collaboration

I make it a priority to help create a positive environment for everyone on the team. We're in this together, and teams that feel comfortable and respect one another are better able to collaborate more effectively, and produce better work as a result.
